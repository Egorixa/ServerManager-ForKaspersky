# ServerManager

Веб-приложение для управления пулом вычислительных серверов. Система позволяет автоматизировать процесс аренды, отслеживать состояние оборудования и управлять жизненным циклом ресурсов в распределенной среде.

---

## Архитектура проекта

Приложение разделено на 4 логических слоя:

- **Domain**: Содержит ядро системы, сущность `ServerEntity`и перечисление состояний `ServerState`.
- **Application**: Содержит интерфейсы репозиториев, сервисов и реализацию бизнес-логики в `ServerService`.
- **Infrastructure**: Реализация доступа к данным через Entity Framework Core.
- **Api**: Входная точка приложения, контроллеры и фоновые задачи для автоматизации процессов.

---

## Ключевые функции

### 1. Жизненный цикл сервера

- Если сервер уже включен (`IsPoweredOn`), он выдается пользователю сразу.
- Если сервер выключен, система переводит его в статус `PoweringOn` на 5 минут.
- Реализован фоновый процесс `AutoReleaseServerJob`, который автоматически освобождает сервер через 20 минут после начала аренды.

### 2. Конкурентность и блокировки

Для предотвращения двойной аренды одного и того же сервера реализована **оптимистичная блокировка**:

- В БД используется токен конкурентности `RowVersion`.
- При попытке одновременного обновления записи база данных генерирует исключение, которое возвращается пользователю как кастомная ошибка `ConcurrentRentException`.

### 3. Поиск и мониторинг

- Поиск свободных серверов по ОС, количеству ядер, объему RAM и диска.
- Возможность узнать, завершился ли процесс включения сервера через эндпоинт `ready`.

### Запуск через Docker

В корне проекта подготовлен `docker-compose.yml`. Для запуска выполните:

```bash
docker-compose up --build
```

API будет доступно по адресу: http://localhost:5000/swagger

### Тестирование

Проект покрыт двумя типами тестов в слое `Tests`:

- Unit-тесты: Проверка корректности смены статусов и логики аренды в ServerService.
- Интеграционные тесты: Проверка работы API-эндпоинтов с использованием WebApplicationFactory

### Дерево проекта:

```
ServerManager/
│
├── docker-compose.yml # Конфигурация для запуска API в контейнере (проброс портов 5000:5000)
├── Dockerfile # Инструкции многоэтапной сборки проекта (multi-stage build)
│
├── Api/ # [Слой представления] Входная точка приложения
│ ├── Api.csproj # Файл конфигурации проекта API
│ ├── Program.cs # Настройка DI-контейнера, Swagger, БД (InMemory) и инициализация (Seed)
│ ├── AutoReleaseServerJob.cs # Фоновая задача (BackgroundService) для авто-освобождения серверов через 20 минут
│ └── Controllers/
│ └── ServersController.cs # REST API эндпоинты (Add, GetAvailable, Rent, Release, IsReady)
│
├── Application/ # [Слой бизнес-логики] Сценарии использования
│ ├── Application.csproj
│ ├── ServerService.cs # Реализация основной логики (аренда, логирование, таймеры)
│ ├── Interfaces/
│ │ ├── IServerRepository.cs # Интерфейс репозитория (Инверсия зависимостей)
│ │ └── IServerService.cs # Интерфейс сервиса серверов
│ └── DTOs/
│ ├── CreateServerDto.cs # Модель для создания нового сервера
│ ├── ServerDto.cs # Модель для возврата данных клиенту
│ └── ServerSearchFilterDto.cs # Модель фильтров для поиска свободных серверов
│
├── Domain/ # [Слой предметной области] Ядро приложения
│ ├── Domain.csproj
│ ├── ServerEntity.cs # Сущность сервера с бизнес-правилами (Rich Model) и полями для конкурентности
│ ├── ServerState.cs # Enum: Available (свободен), PoweringOn (включается), Rented (арендован)
│ └── ConcurrentRentException.cs # Кастомное исключение при оптимистичной блокировке
│
├── Infrastructure/ # [Слой доступа к данным] Работа с БД (EF Core)
│ ├── Infrastructure.csproj
│ ├── AppDbContext.cs # Контекст базы данных Entity Framework
│ ├── ServerConfiguration.cs # Настройки маппинга и указание RowVersion как ConcurrencyToken
│ └── ServerRepository.cs # Реализация интерфейса репозитория
│
└── Tests/ # [Слой тестирования] xUnit + Moq
├── Tests.csproj
├── ServerApiTests.cs # Интеграционные тесты (TestServer + InMemoryDb) для проверки эндпоинтов
└── ServerServiceTests.cs # Юнит-тесты бизнес-логики (проверка смены статусов и времени)

```
